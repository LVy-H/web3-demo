# Frontend — ZK Ballot & Lottery

React 19 web interface for the ZK Ballot & Lottery PoC.

## Stack

| Library | Version | Purpose |
|---------|---------|---------|
| React | 19 | UI framework |
| Vite | 8 | Build tool |
| TypeScript | 5.9 | Type safety |
| Wagmi | 3 | React hooks for Ethereum |
| Viem | 2 | Ethereum client |
| TailwindCSS | 4 | Styling |
| @tanstack/react-query | 5 | Async state management |
| @semaphore-protocol/proof | 4 | ZK proof generation |
| @semaphore-protocol/identity | 4 | Anonymous identity keypairs |
| @semaphore-protocol/group | 4 | Off-chain Merkle tree mirror |
| @metamask/sdk | 0.33 | MetaMask wallet connector |

## Development

```bash
npm install
npm run dev        # http://localhost:5173
npm run build      # Production build
npm run lint       # ESLint
```

## Environment

The app expects a running Hardhat node at `http://127.0.0.1:8545` with the `ZkVotingLottery` contract deployed at `0xCf7Ed3AccA5a467e9e704C703E8D87F634fB0Fc9`.

See the root [README](../README.md) for the full setup guide.

## Key Files

| File | Description |
|------|-------------|
| `src/App.tsx` | Main component — wallet, identity, vote, claim UI |
| `src/config.ts` | Wagmi config, chain definitions, contract address |
| `src/main.tsx` | React root with WagmiProvider and QueryClientProvider |
| `src/ZkVotingLottery.json` | Contract ABI (auto-generated by Hardhat, do not edit) |
| `tests/e2e.spec.ts` | Playwright end-to-end tests |
